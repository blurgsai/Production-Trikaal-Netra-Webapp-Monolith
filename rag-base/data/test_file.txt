Here are the quick, step-by-step commands to test the dynamic resource refresh behavior. 

You can perform this test using your terminal (with `curl` and Python one-liners). Make sure your services (`make test-ui` or equivalent) are running.

---

### Step 0: Generate an Authentication Token
Run this command to generate a JWT token for the user `pavan`:
```bash
TOKEN=$(python -c "import jwt, time; print(jwt.encode({'sub': 'pavan', 'role': 'user', 'iat': int(time.time())}, 'jwt-secret-key', algorithm='HS256'))") && echo "Token: $TOKEN"
```

---

### Step 1: Create a New Unique Document Locally
Create a text file containing unique info that the LLM could not know beforehand:
```bash
echo "The Cosmic Energy Grid X-Ray is a top-secret containment system developed in the year 2099 by the Orion Sector." > data/Cosmic_Grid_XRay.txt
```

---

### Step 2: Upload the Document to MinIO
Upload the file directly to your MinIO storage bucket using the Python client:
```bash
python -c "from core.clients.minio_client import MinioClient; client = MinioClient(); client.client.fput_object(client.bucket_name, 'Cosmic_Grid_XRay.txt', 'data/Cosmic_Grid_XRay.txt')"
```

---

### Step 3: Create a Session & Ask Question (Expect No Answer)
1. **Create a new chat session**:
   ```bash
   SESSION_ID=$(curl -s -X POST http://localhost:8000/sessions \
     -H "Authorization: Bearer $TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"title": "Resource Refresh Test"}' | python -c "import sys, json; print(json.load(sys.stdin)['session_id'])") && echo "Session ID: $SESSION_ID"
   ```

2. **Ask about the Cosmic Grid** (since it hasn't been ingested into Chroma yet, it should fail or state that it doesn't know):
   ```bash
   curl -N -X POST http://localhost:8000/stream \
     -H "Authorization: Bearer $TOKEN" \
     -H "Content-Type: application/json" \
     -d "{\"session_id\": \"$SESSION_ID\", \"message\": \"What is the Cosmic Energy Grid X-Ray?\"}"
   ```
   *(Expect response: "I could not find any specific information..." or similar fallback)*

---

### Step 4: Add the Document Using the Global Endpoint
Trigger the ingestion. This downloads the file from MinIO, indexes it into Chroma under `global_documents`, registers it in MongoDB, and marks the LLM client's resources as stale:
```bash
curl -X POST http://localhost:8000/add-global-documents \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d "{\"file_path\": \"Cosmic_Grid_XRay.txt\", \"file_name\": \"Cosmic_Grid_XRay.txt\", \"description\": \"Top-secret grid from 2099\", \"session_id\": \"$SESSION_ID\"}"
```
*(Wait 3 seconds for the indexes and webhook notifications to propagate)*

---

### Step 5: Ask Again in the Same Session (Expect Successful Retrieval)
Ask the same question again. Because the cache was invalidated, the LLM will re-fetch the resource metadata from the MCP server, see the new document registry, and query it:
```bash
curl -N -X POST http://localhost:8000/stream \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d "{\"session_id\": \"$SESSION_ID\", \"message\": \"What is the Cosmic Energy Grid X-Ray?\"}"
```
*(Expect response: "The Cosmic Energy Grid X-Ray is a top-secret containment system developed in the year 2099 by the Orion Sector.")*
